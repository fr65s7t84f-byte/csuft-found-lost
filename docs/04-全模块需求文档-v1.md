# 校园失物招领系统全模块需求文档 v1

## 1. 范围说明
本版本覆盖以下模块并形成可联调后端初版：
- 用户与认证模块
- 失物模块
- 招领模块
- 用户发布与详情模块
- 匹配推荐模块
- 公告模块
- 管理端模块（看板、认领审核、用户管理、公告管理、物品审核）
- 数据可视化模块（积分榜、热点图、聚合接口）
- 图片智能解析模块

## 2. 业务角色
- 普通用户：登录注册、发布失物/招领、查看匹配、撤回、看公告与热力图。
- 管理员：审核认领、审核物品、维护公告、管理用户、查看看板。

## 3. 功能需求清单

### 3.1 认证模块
- 登录：账号+密码+角色校验。
- 注册：手机号注册普通用户。
- 返回统一认证信息：token、role、name、avatarUrl。

### 3.2 失物/招领模块
- 列表查询（分页、关键词、分类、地点、标签、状态、审核状态）。
- 发布信息。
- 修改信息。
- 删除信息。

### 3.3 用户发布模块
- 查询本人发布列表（可按类型过滤）。
- 查看物品详情。
- 撤回发布（状态置为关闭）。

### 3.4 匹配模块
- 查看某条记录的匹配候选TopN。
- 输出匹配分数与可解释原因。

### 3.5 公告模块
- 用户端今日公告聚合（今日失物、今日拾物、紧急寻物、重要通知）。
- 管理端公告增删改查（初版实现增改查）。

### 3.6 管理端模块
- 看板概览（总登记量、找回率、活跃用户、趋势和分类统计）。
- 认领审核。
- 用户状态管理（active/frozen）。
- 物品审核（lost/found）。

### 3.7 可视化与智能模块
- 积分榜接口。
- 热点图接口。
- campus-insights 聚合接口。
- 图片智能解析与匹配预览接口。

## 4. 非功能需求
- 统一响应结构、统一异常处理。
- 基于数据库持久化（初版使用 H2 MySQL 模式，保留 MySQL 兼容DDL）。
- 模块化分层（controller/service/repository/entity）。
- 可扩展到真实AI服务与MySQL生产库。

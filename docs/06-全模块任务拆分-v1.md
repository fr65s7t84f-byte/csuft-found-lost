# 校园失物招领系统全模块任务拆分 v1

## 1. 迭代目标
在一个迭代内完成“全模块文档 + 后端初版 + 数据库 + 联调接口”。

## 2. 工作分解结构（WBS）

### 阶段A：需求与设计
- A1：全模块需求清单梳理
- A2：接口契约与字段冻结
- A3：架构分层与数据库设计评审

### 阶段B：数据库与基础能力
- B1：落地 `schema.sql`
- B2：落地 `data.sql`
- B3：统一响应与统一异常处理

### 阶段C：业务模块开发
- C1：认证模块（登录、注册）
- C2：失物/招领模块（增删改查）
- C3：用户发布与详情模块
- C4：匹配模块（记录匹配、预览匹配）
- C5：公告模块（今日公告聚合）
- C6：管理端模块（看板、审核、公告、用户）
- C7：统计模块（积分榜、热点图、聚合）

### 阶段D：联调与验证
- D1：后端编译通过
- D2：前端关键接口连通性验证
- D3：文档与代码一致性检查

## 3. 里程碑
- M1：文档冻结
- M2：数据库与基础框架完成
- M3：全模块接口完成
- M4：编译通过并可联调

## 4. 风险控制
- 鉴权缺失风险：初版采用简化登录，后续补JWT
- AI稳定性风险：先规则模拟再替换真实模型
- 数据一致性风险：事务包裹关键更新接口

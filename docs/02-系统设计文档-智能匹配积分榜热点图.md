# 校园失物招领系统设计文档（智能匹配 + 积分榜 + 热点图）

## 1. 设计目标
- 为前端新增页面提供稳定后端接口。
- 将“AI解析”“匹配引擎”“统计展示”分层，降低耦合。
- 支持后续从内存实现平滑迁移到数据库 + AI网关。

## 2. 总体架构

### 2.1 架构分层
- Controller层：HTTP接口与参数校验。
- Service层：业务编排（AI解析、匹配打分、统计聚合）。
- Repository层（初版以内存替代）：提供候选数据源。
- Model/DTO层：入参与出参契约。

### 2.2 模块划分
- `IntelligenceController`
  - 图片解析
  - 匹配预览
- `StatisticsController`
  - 积分榜
  - 热点图
  - 聚合接口 campus-insights
- `IntelligenceService`
  - 图像结果结构化
  - 匹配评分与排序
- `StatisticsService`
  - 榜单数据组织
  - 地图点位聚合

## 3. 关键设计

### 3.1 图片解析结果结构（统一输出）
- `category`
- `subCategory`
- `tags[]`
- `ocrTexts[]`
- `brand`
- `color`
- `confidence`
- `riskLevel`

### 3.2 匹配评分策略（初版）
- 基础分 40
- 类别一致 +20
- 地点一致 +15
- 标签交集每个 +6，上限 +24
- 标题相似加分 +0~10
- 总分上限 99，按分数降序返回

### 3.3 统计聚合模型
- `map`: 学校名称、静态图URL、来源、更新时间
- `leaderboard[]`: 排名、积分、发布/归还统计
- `hotspots[]`: 点位坐标百分比、类型、权重、拾取次数

## 4. API设计

### 4.1 `POST /api/intelligence/analyze-image`
- Request
```json
{
  "imageUrl": "https://...",
  "itemType": "lost",
  "locationHint": "图书馆",
  "description": "黑色双肩包"
}
```
- Response
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "category": "箱包",
    "subCategory": "双肩包",
    "tags": ["黑色", "帆布", "双肩包"],
    "ocrTexts": ["CSUFT"],
    "brand": "unknown",
    "color": "黑色",
    "confidence": 0.87,
    "riskLevel": "low"
  }
}
```

### 4.2 `POST /api/intelligence/match-preview`
- Request
```json
{
  "itemType": "lost",
  "title": "黑色双肩包",
  "category": "箱包",
  "location": "图书馆",
  "tags": ["黑色", "帆布", "书包"]
}
```
- Response
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 2001,
      "title": "黑色帆布书包",
      "location": "图书馆",
      "score": 93,
      "reasons": ["同类物品", "地点接近", "共同标签：黑色、帆布"]
    }
  ]
}
```

### 4.3 `GET /api/statistics/campus-insights`
- Response
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "map": {},
    "leaderboard": [],
    "hotspots": []
  }
}
```

## 5. 数据持久化演进设计
- V1（当前）：内存数据，快速联调。
- V2：MySQL（用户、物品、匹配记录、积分记录）+ Redis 缓存热点。
- V3：向量库（图像/文本 embedding）与离线召回索引。

## 6. 异常与容错设计
- 参数缺失返回业务错误码与错误信息。
- AI解析失败时返回兜底标签（`unknown`）而非500。
- 匹配无结果返回空数组，不抛错。

## 7. 安全与合规设计
- 不在接口直接返回完整敏感证件号。
- 统一网关层做限流和鉴权（初版预留）。
- 后续补充审计日志字段：`requestId`, `operator`, `traceId`。

## 8. 部署建议（初版）
- 单体 Spring Boot 服务，端口 `8080`。
- 前端通过 Vite 代理或网关转发到 `/api`。
- 日志级别 `INFO`，匹配明细打 `DEBUG`。

# 校园失物招领系统全模块设计文档 v1

## 1. 架构设计
- 技术栈：Spring Boot + Spring Web + Spring Data JPA + H2(MySQL mode)
- 分层：
  - Controller：接口层
  - Service：业务编排层
  - Repository：数据访问层
  - Entity/Model/DTO：数据模型层

## 2. 模块设计

### 2.1 认证模块
- `AuthController` + `AuthService`
- 登录逻辑：账号存在校验、密码校验、角色校验、返回简易token
- 注册逻辑：账号唯一约束、用户默认角色和状态初始化

### 2.2 失物/招领模块
- `LostController`、`FoundController` 复用 `ItemService`
- 核心实体：`tb_item`
- 关键逻辑：筛选分页、发布、更新、删除

### 2.3 用户发布与详情模块
- `UserItemController`
- 关键逻辑：按发布人查询、详情查询、撤回

### 2.4 匹配模块
- `ItemService.matches`（记录级匹配）
- `IntelligenceService.match-preview`（请求级匹配）
- 评分策略：类别、地点、标签交集、标题字符相似度加权

### 2.5 公告模块
- `AnnouncementController` + `AnnouncementService`
- 聚合逻辑：失物/招领最新记录 + 系统公告，拼装前端今日公告视图

### 2.6 管理端模块
- `AdminController` + `AdminService`
- 看板逻辑：总量、认领率、活跃用户、趋势与分类统计
- 审核逻辑：认领审核、物品审核、用户状态更新、公告维护

### 2.7 数据可视化模块
- `StatisticsController` + `StatisticsService`
- 榜单来源：用户信用分排序
- 热点来源：`tb_hotspot`
- 聚合接口：`campus-insights`

## 3. 数据库设计
- `tb_user`：账户、角色、状态、信用分
- `tb_item`：失物/招领统一建模
- `tb_claim`：认领记录
- `tb_notice`：公告
- `tb_hotspot`：热点点位
- DDL见：`backend/src/main/resources/schema.sql`
- 初始数据见：`backend/src/main/resources/data.sql`

## 4. 关键接口契约
- 统一响应：`{ code, message, data }`
- 分页结构：`{ records, total, pageNum, pageSize }`
- 时间格式：`yyyy-MM-dd HH:mm`

## 5. 可扩展设计
- 鉴权：后续接入 JWT 拦截器
- AI：替换规则模拟为真实大模型网关
- 持久化：从H2平滑迁移到MySQL
- 匹配：接入向量检索和反馈学习
